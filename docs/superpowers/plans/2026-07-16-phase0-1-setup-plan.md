# Phase 0 and Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task by task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the SpendWeiss backend project with `uv`, add three mock data files, and build a hand rolled ReAct loop that talks to Groq directly and calls two data lookup tools, with no agent framework involved.

**Architecture:** A `uv` managed Python project under `backend/`. Static JSON data lives in `backend/data/`. Phase 1 code lives in `backend/phase1_raw_react/`, split into `tools.py` (pure data lookup functions) and `agent.py` (the loop, the system prompt, and the interactive entry point).

**Tech Stack:** Python 3.12, `uv`, `groq` Python client, `python-dotenv`.

## Global Constraints

- All prose (docs, journal entries, commit messages) is British English, with no em dashes.
- Code carries explanatory comments in British English (this reverses an earlier "no comments" rule; see `JOURNAL.md`, 2026-07-16).
- No automated test suite for this plan. Verification steps are manual: run a command, read the output, confirm it matches what is described.
- Do not run `git add`, `git commit`, or `git push` at any point in this plan. All files are left in the working tree, unstaged, for the user to review and commit themselves.
- Model id for all Groq calls: `llama-3.3-70b-versatile`.
- Phase 1 does not use Groq's native `tools=` parameter. The tool calling protocol is defined by hand in the system prompt as a JSON schema.

### Task 3: Build the tool functions

**Files:**
- Create: `backend/phase1_raw_react/tools.py`

**Interfaces:**
- Consumes: `backend/data/cards.json`, `backend/data/offers.json` (from Task 2).
- Produces: `check_card_rewards(category: str) -> list[dict]`, returning a list of `{"card_id", "card_name", "reward_rate"}` dicts sorted highest reward rate first, only including cards that define a rate for that category. `check_offers(merchant: str) -> list[dict]`, returning the raw offer records whose `merchant` field case insensitively contains the given string. Both are imported directly by `agent.py` in Task 4.

- [ ] **Step 1: Write `backend/phase1_raw_react/tools.py`**

```python
import json
from pathlib import Path

DATA_DIR = Path(__file__).resolve().parent.parent / "data"


def load_cards():
    with open(DATA_DIR / "cards.json") as data_file:
        return json.load(data_file)


def load_offers():
    with open(DATA_DIR / "offers.json") as data_file:
        return json.load(data_file)


def check_card_rewards(category):
    cards = load_cards()
    matches = []
    for card in cards:
        rate = card["reward_rates"].get(category)
        if rate is not None:
            matches.append({
                "card_id": card["id"],
                "card_name": card["name"],
                "reward_rate": rate,
            })
    matches.sort(key=lambda entry: entry["reward_rate"], reverse=True)
    return matches


def check_offers(merchant):
    offers = load_offers()
    needle = merchant.lower()
    return [offer for offer in offers if needle in offer["merchant"].lower()]
```

### Task 4: Build the ReAct loop

**Files:**
- Create: `backend/phase1_raw_react/agent.py`

**Interfaces:**
- Consumes: `check_card_rewards`, `check_offers` from `backend/phase1_raw_react/tools.py` (Task 3).
- Produces: a runnable script, `uv run phase1_raw_react/agent.py` from inside `backend/`, that reads `GROQ_API_KEY` from the environment (via `.env`), prompts for a purchase description, runs the loop, prints every step, and prints a final recommendation.

- [ ] **Step 1: Write `backend/phase1_raw_react/agent.py`**

```python
import json
import os

from dotenv import load_dotenv
from groq import Groq

from tools import check_card_rewards, check_offers

load_dotenv()

MODEL = "llama-3.3-70b-versatile"
MAX_ITERATIONS = 6

SYSTEM_PROMPT = """You are SpendWeiss, an assistant that recommends the best card for a purchase.

You have two tools available:

1. check_card_rewards
   Args: {"category": "<one of groceries, dining, travel, online_shopping, fuel, other>"}
   Returns a list of cards with their reward rate for that category, highest first.

2. check_offers
   Args: {"merchant": "<merchant name>"}
   Returns a list of active offers that match the given merchant.

Reply with exactly one JSON object per turn, and nothing else. No prose outside the JSON.

To call a tool, reply with:
{"action": "check_card_rewards", "args": {"category": "..."}}
or
{"action": "check_offers", "args": {"merchant": "..."}}

Once you have enough information, reply with:
{"action": "final_answer", "answer": "<your recommendation and reasoning>"}
"""

TOOLS = {
    "check_card_rewards": check_card_rewards,
    "check_offers": check_offers,
}


def run_query(client, purchase_description):
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": purchase_description},
    ]

    retried = False

    for _ in range(MAX_ITERATIONS):
        completion = client.chat.completions.create(
            messages=messages,
            model=MODEL,
        )
        raw_content = completion.choices[0].message.content
        print(f"\nModel: {raw_content}")

        try:
            action = json.loads(raw_content)
        except json.JSONDecodeError:
            if retried:
                print("\nGiving up: the model failed to produce valid JSON twice.")
                return
            retried = True
            messages.append({"role": "assistant", "content": raw_content})
            messages.append({
                "role": "user",
                "content": "That was not valid JSON. Reply with exactly one JSON object as instructed.",
            })
            continue

        action_name = action.get("action")

        if action_name == "final_answer":
            print(f"\nRecommendation: {action['answer']}")
            return

        tool_fn = TOOLS.get(action_name)

        if tool_fn is None:
            if retried:
                print(f"\nGiving up: unknown action '{action_name}' requested twice.")
                return
            retried = True
            messages.append({"role": "assistant", "content": raw_content})
            messages.append({
                "role": "user",
                "content": f"Unknown action '{action_name}'. Use check_card_rewards, check_offers, or final_answer.",
            })
            continue

        tool_args = action.get("args", {})
        result = tool_fn(**tool_args)
        print(f"Tool result: {result}")

        messages.append({"role": "assistant", "content": raw_content})
        messages.append({"role": "user", "content": json.dumps({"tool_result": result})})

    print("\nGiving up: reached the maximum number of reasoning steps.")


def main():
    client = Groq(api_key=os.environ["GROQ_API_KEY"])

    print("SpendWeiss Phase 1. Describe a purchase, or press Ctrl+C to quit.")
    while True:
        try:
            purchase_description = input("\nWhat's the purchase? ")
        except KeyboardInterrupt:
            print("\nGoodbye.")
            break

        if not purchase_description.strip():
            continue

        run_query(client, purchase_description)


if __name__ == "__main__":
    main()
```
